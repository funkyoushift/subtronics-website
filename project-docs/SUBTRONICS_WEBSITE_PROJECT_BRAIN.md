Secondary conversions:
- Contact form
- Service inquiries
- Custom PC consultations

## Site Rules

### Rule 1 — Always deliver a finished build
Whenever website changes are requested, implement the work and return an updated downloadable build.
Do not stop at instructions unless guidance-only is explicitly requested.

### Rule 2 — Full-site audit after every major addition/removal
After major changes, audit:
- navigation
- footer
- internal linking
- sitemap.xml
- schema
- llms.txt
- robots.txt
- canonicals
- page titles
- meta descriptions

### Rule 3 — No orphan pages
Every important page must receive internal links from relevant pages.

### Rule 4 — Internal linking is required
New service pages should typically receive links from:
- homepage
- services hub
- related service pages
- footer if appropriate
- relevant help / FAQ content when useful

### Rule 5 — No redundant or cannibalizing pages
Do not create multiple pages targeting the same exact intent unless there is a clear reason.

### Rule 6 — Clean navigation only
Unfinished, low-conversion, or off-strategy pages should be hidden from primary navigation.

### Rule 7 — SEO basics on every page
Every indexable page should have:
- title tag
- meta description
- H1
- clean heading structure
- relevant internal links
- alt text on meaningful images
- canonical URL where appropriate

### Rule 8 — Schema must stay current
Whenever services or site structure change, update relevant schema such as:
- LocalBusiness / ComputerRepair
- Service
- FAQPage
- OfferCatalog where useful

### Rule 9 — AI pointers must stay current
Update:
- llms.txt
- sitemap.xml
- structured data
- machine-readable service summaries

### Rule 10 — Image optimization
Use compressed images. Prefer WebP where practical. Include descriptive alt text.

### Rule 11 — Changelog required
Each release should include a changelog or audit note file.

### Rule 12 — Automatic correction of rule violations
If a rule violation is noticed during a site update, it should be corrected during the same pass rather than merely described.
